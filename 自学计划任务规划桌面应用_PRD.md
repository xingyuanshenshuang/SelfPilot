# 自学计划任务规划桌面应用（修订版）

**版本**: 1.0  
**日期**: 2026-06-15  
**状态**: 修订版  

---

## 1. 产品概述

一款面向自学者、学生、考证人群的桌面任务规划工具。核心价值在于支持多层级目标自动拆解与手动任务混合管理，进度自动向上汇总，并提供可视化的完成情况与鼓励机制。

---

## 2. 用户角色

唯一角色：个人用户（单机使用，无需登录）。

---

## 3. 信息架构与布局

- **左右结构**：
  - **左侧导航栏**：承载主要功能模块的切换入口。
  - **右侧内容视图**：根据左侧选中的模块，展示对应的操作界面。

---

## 4. 功能需求

### 4.1 左侧导航栏（固定模块）

包含以下菜单项（按顺序）：

1. **目标总览**（Dashboard） – 核心进度仪表盘
2. **目标树** – 查看所有目标及关联的计划/任务层级
3. **日历视图** – 以日历形式查看每日任务及完成情况
4. **数据统计** – 图表展示进度趋势
5. **鼓励语库** – 管理预设及自定义鼓励语
6. **设置** – 系统参数配置（仅此一处设置入口，右上角不再重复）

### 4.2 右侧视图 – 各模块详情

#### 模块一：目标总览（默认首页）

**展示内容**：

- 所有目标列表及其完成百分比（环形或进度条）
- **今日待办**（包含今日应完成的任务 + 逾期未完成任务，逾期任务标红）
- **逾期任务**：截止日期早于今日且未完成的任务，支持快速补完成
- 快捷操作：完成/跳过/补完成（数量型任务完成时需填写实际完成量）

**鼓励语**：当日首次完成任务后，弹窗或顶部提示一句鼓励语（随机从语库中抽取，且考虑连续完成天数）。

#### 模块二：目标树

**简化后的层级结构**（自上而下）：

```
目标（一级）
└─ 阶段（二级，可选，用户手动添加用于分组）
└─ 任务（三级，实际执行单元）
```

**说明**：

- 不再强制生成“计划/月计划/周计划”中间层级。自动拆解时，直接在目标下生成带日期的任务（任务层级），任务按日期在日历视图和今日待办中组织。
- 手动任务可挂在任意目标或阶段下，也可作为“独立任务”（显示在根节点“临时任务”下）。

**操作功能**：

- **创建目标**：输入名称、截止日期、总量（单位可选“个/页/小时”），总量可为0（表示按任务数计算进度）。
- **自动拆解**：点击目标旁的【自动拆解计划】→ 系统按截止日期剩余天数（不含今天，剩余天数≥1），将总量平均分配到每天的任务（余数分配到前几天）。每个任务记录“计划数量”和“单位”。用户可手动修改每日的任务计划数量。
- **手动添加任务**：在任意节点上右键或点击【+】，选择任务类型（阶段/任务），填写名称、计划日期（支持单天或周期：每日/每周几/每月几号）、计划数量（可选，默认1）。
- **编辑/删除**：支持修改截止日期、计划数量、关联关系。删除某一任务时，其子任务（阶段下的任务）转为独立任务（提示用户是否级联删除，默认保留独立）。
- **标记完成（支持部分完成）**：
  - 若任务计划数量 = 1：勾选复选框即为完成。
  - 若任务计划数量 > 1：点击【完成】按钮弹出窗口，输入**实际完成数量**（0 ~ 计划数量）。系统按实际完成量 / 计划数量 计算该任务的完成百分比，并向上汇总至父级目标和阶段。
  - 任务状态：✅ 100%完成，🟡 部分完成（1%~99%），🔲 未完成（0%）。
- **跳过**：标记该任务为“已跳过”（不计入完成，也不影响后续任何自动调整）。跳过后的任务可在历史中补完成。
- **补完成**：选择历史日期，找到对应任务，按标记完成流程输入实际完成量。补完成**不会重新计算未完成任务的计划数量**，仅更新历史完成记录和累计进度。

#### 模块三：日历视图

- 以月/周/日网格展示。
- 每个单元格内显示该日期下所有任务（自动拆解的任务 + 周期性手动任务）：
  - 任务名称及完成状态（✅ / 🟡 / 🔲 / ⚠️逾期）
- 点击日期展开当天详细任务列表，支持批量标记完成/跳过（批量时数量型任务需逐个输入，或设为“全部按计划完成”）。

#### 模块四：数据统计（图表）

- **切换维度**：
  - 各目标完成百分比（柱状图）
  - 时间趋势（折线图）：近7天/30天完成任务数量（数量型任务按完成数量统计，布尔型任务完成计1）
- **日历热力图**：颜色深浅表示每日完成任务比例（完成任务量 / 当日应有任务总量）。
- **预测完成时间**（加分项）：基于过去7天的平均完成速度，预测每个目标是否能按期完成，并给出建议（如“需提高速度”或“可提前完成”）。

#### 模块五：鼓励语库

- **预设清单**（不少于10条，例如“今天又进步了！”“连续打卡3天，好样的！”）。
- **用户自定义**：支持添加/编辑/删除自己的鼓励语。
- **个性化规则**：
  - 连续完成标准：**每天至少完成一个任务**（不论是否有计划任务；若当天没有任何任务，则不中断也不计入连续）。
  - 连续1天：普通语库
  - 连续3天：优先抽取“进阶鼓励语”
  - 连续7天：特殊高亮鼓励语
  - 完成全部目标：庆祝语

#### 模块六：设置

- **行为设置**：
  - 跳过任务的处理：仅一种模式（标记跳过，不影响后续计划）。提供独立的 **“重新规划剩余时间”** 按钮（位于目标详情页），用户手动触发后，将目标下所有未完成的任务（不含跳过的）按剩余天数重新平均分配计划数量。
- **数据管理**：导出/导入数据（本地JSON或SQLite备份）。导入时默认合并（若ID冲突则提示：覆盖/跳过/重命名）。
- **外观**：浅色/深色主题（可选）。

---

## 5. 输入输出汇总

| 类型 | 内容 |
|------|------|
| **输入** | 目标信息（名称、截止日、总量）、手动任务信息（名称、计划日期、计划数量）、每日完成数量、跳过/补完成标记、自定义鼓励语、系统设置选项 |
| **输出** | 自动拆解的任务列表（按日期分配）、各层级进度百分比（基于完成量/计划总量）、日历视图、统计图表、个性化鼓励语、本地存储文件 |

---

## 6. 非功能需求

- **平台**：Windows / macOS 桌面应用
- **数据存储**：本地 SQLite 或 JSON 文件，无需网络
- **性能**：启动时间 < 3 秒，任务操作响应 < 0.5 秒
- **可扩展性**：代码结构预留云端同步接口（后续版本）
- **数据模型**：每个任务/目标使用 UUID，父子关系通过 parent_id + path 存储

---

## 7. 异常处理

- **截止日期为空或已过期**：提示用户修改，不允许自动拆解。
- **自动拆解时剩余天数为0**：提示至少留出1天。
- **用户删除某一包含已完成子任务的阶段/目标**：提示“子任务已完成，删除后统计数据会变化”，并确认是否继续。
- **跨天补完成导致重复标记**：系统禁止对已完成的任务再次标记完成（可修改实际完成数量，但需校验不超过计划数量）。
- **导入数据时ID冲突**：弹出对话框让用户选择：覆盖现有任务、跳过该记录、或自动生成新ID（合并）。
